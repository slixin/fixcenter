/client/connect

Used to build the connection between FIX Client to Fix server.

  • METHOD: POST
  • PARAMETERS:
    • senderid - Sender ID of Fix Session
    • targetid - Target ID of Fix Session
    • host - FIX Server name /IP
    • port - FIX Server port
    • version - FIX version, 4.2, 4.3, 4.4, 5.0
    • user - optional, used by FIX Cloud UI.
  • RETURN:
    • ​Pass
      • status - boolean
      • client_id - FIX session ID, i.e. FIXSERVER|8000|4.4-SENDER-TARGET
      • listener_port - only when user parameter is used.
    • Fail
      • status - boolean
      • error - string
  • EXAMPLE:
    • ​Request:
      • /client/connect
        • senderid=SENDER
        • targetid=TARGET
        • host=192.168.1.1
        • port=8000
        • version=4.4
    • Response:
      • { status: true, client_id: FIXSERVER|8000|4.4-SENDER-TARGET, listener_port: null }

/client/send/bulk

Used to bulk sending FIX messages

  • METHOD: POST
  • PARAMETERS:
    • client - Fix Session ID which return by /client/connect, i.e. FIXSERVER|8000|4.4-SENDER-TARGET
    • message - JSON format FIX message, see detail in Example.
    • amount - Total amount for bulk sending, i.e. 3000
    • tps - transaction per seconds for bulk sending, i.e. 100
    • reference_bulk_id - When the bulk sending is based on previous bulk sending, refer the previous bulk_id
  • RETURN:
    • status - boolean
    • id - Bulk execution ID, it is a GUID number.

/client/send/bulk/result

Used to get the bulk sending result.

  • METHOD: POST
  • PARAMETERS:
    • client - Fix Session ID which return by /client/connect, i.e. FIXSERVER|8000|4.4-SENDER-TARGET
    • id - Bulk execution ID, it is a GUID number.
    • detail - boolean, whether to show details of Bulk sending.
  • RETURN:
    • detail is True
      • status - boolean
      • count - Total count for orders
      • orders[] - return orders array
        • order_initial_id
        • order_status
        • order_messages[]
          • key
          • messages[]
    • detail is False
      • status - boolean
      • count - Total count for sending messages
  • EXAMPLE:

 

/client/send/bulk/result/detail

Used to get the bulk sending result by order ID

  • METHOD: POST
  • PARAMETERS:
    • client - Fix Session ID which return by /client/connect, i.e. FIXSERVER|8000|4.4-SENDER-TARGET
    • bulk_id - Bulk execution ID, it is a GUID number.
    • order_id - order ID for the order in the bulk execution.
  • RETURN:
    • Pass:
      • status - boolean
      • order - specific order return
        • order_initial_id
        • order_status
        • order_messages[]
          • key
          • messages[]

      • status - boolean
      • error - string
