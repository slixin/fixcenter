What is FIX Centre?

The Financial Information eXchange (FIX) protocol is an electronic communications protocol initiated in 1992 for international real-time exchange of information related to the securities transactions and markets. FIX Centre is designed for developers and testers to simulating the FIX Client application and FIX Server for verifying their FIX products.

How to enter FIX message

FIX Centre use JSON format data as FIX message format

There are 4 ways to enter the FIX messages: Tree / Form / Code / Text

Even you have no idea of JSON format, you can use Tree or Form mode to edit your message.

Repeating group

When you are going to create some FIX message includes multiple-leg or repeating group

You can use { value: xxx, groups: [{},{}]} in the JSON message.

Sequential message

When you are going to sending bulk message with some sequential messages, for example: Create order, then amend it, then cancel it.

You can enter the array of JSON messages.

Value placeholder

There are 2 scenarios you'd like to replace the value in the FIX message template to other value.

  • Response a message, i.e. Cancel a create OK order
  • Enter a sequential messages, i.e. Create order, then Amend it

You can use ":[tag number]" in the tag value of the template.

Random value for tag

Sometimes, you'd like to use some random pick value for some tag, for example: BuySellSide (54), Price (44)

You can use array in the JSON directly, it will randomly pick one value in the array
